Polyaxon allows to schedule dask clusters via dask-kubernetes natively.

## Deploy the DaskCluster operator

Before you can use the daskcluster runtime, you need to make sure that DaskCluster operator and the CRD (custom resource definition)
are deployed in your cluster.

## Enable the operator

To be able to schedule distributed jobs with the DaskCluster operator, you need to enable the operator in your deployment config.

> You need to enable the operator in Polyaxon CE deployment or Polyaxon Agent deployment:

```yaml
operators:
  daskcluster: true
```

## Create a component with the daskcluster runtime

Once you have the DaskCluster operator running on a Kubernetes namespace managed by Polyaxon,
you can check the specification for creating components with the daskcluster runtime:

```bash
version: 1.1
kind: component
run:
  kind: daskcluster
  ...
```

For more details about the specification for creating daskcluster runtime, please check please check this
[section](/docs/workload/clusters/dask-clusters/).

## Run the distributed job


Running components with the daskcluster runtime is similar to running any other component:

```bash
polyaxon run -f manifest.yaml -P ...
```

## View a running operation on the dashboard

After running an operation with this component, you can view it on the Dashboard:

```bash
polyaxon ops dashboard
```

or

```bash
polyaxon ops dashboard -p [project-name] -uid [run-uuid] -y
```

## Stop a running operation

To stop a running operation with this component:

```bash
polyaxon ops stop
```

or

```bash
polyaxon ops stop -p [project-name] -uid [run-uuid]
```

## Run the job using the Python client

To run this component using Polyaxon Client:

```python
from polyaxon.client import RunClient

client = RunClient(...)
client.create_from_polyaxonfile(polyaxonfile="path/to/file", ...)
```
