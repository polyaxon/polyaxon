# Secrets and Connections

Sandbox sessions should use the same credential model as the rest of your Polyaxon workloads. Store credentials in Polyaxon connections or Kubernetes-backed secrets, then mount only the connections required by the run.

## Use Connections

Connections are managed by administrators and can represent storage, registries, Git credentials, databases, service accounts, and other external systems. See [connections setup](/docs/setup/connections/overview/) for available connection types.

## Pass Connections to a Run

Use presets or the Polyaxonfile environment and init sections to make connections available to the sandbox. Prefer presets for team-wide defaults:

```bash
polyaxon run -f sandbox.yaml -p owner/project --presets dev-connections
```

## Git over SSH

For private repositories, use an [SSH connection](/docs/setup/connections/ssh/) instead of copying keys into the image or notebook.

## Security Guidance

- Mount only the credentials required by the sandbox.
- Prefer short-lived or scoped credentials.
- Avoid printing secrets in logs or notebooks.
- Stop sandbox runs when access is no longer needed.
- Use [RBAC](/docs/administration/rbac/) and project permissions to control who can start and access sandboxes.
