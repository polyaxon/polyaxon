# Polyaxon Overview

Polyaxon is an **open-source ML & AI platform** ([GitHub](https://github.com/polyaxon)) that helps teams build, train, and deploy machine learning models at scale. All platform features are natively integrated to accelerate the development workflow. Polyaxon is open, self-hostable, and extensible ([_why polyaxon?_](/#why-polyaxon)).

## Experiment Tracking [#tracking]

[Experiment Tracking](/docs/tracking) captures every detail of your ML experiments — hyperparameters, metrics, code versions, data lineage, and output artifacts. Compare runs side-by-side and visualize training curves in real time.

- Log metrics, parameters, and artifacts with the Python SDK
- Compare and visualize runs with built-in dashboards
- Track data lineage and code versions for full reproducibility
- Built-in support for TensorBoard, Plotly, and custom visualizations

## Orchestration [#orchestration]

[Orchestration](/docs/orchestration) lets you compose multi-step ML workflows with DAGs, run hyperparameter sweeps, and schedule recurring pipelines — all with a declarative YAML specification.

- Build DAGs with the [flow engine](/docs/orchestration) for multi-step pipelines
- Run hyperparameter optimization with grid, random, Bayesian, and [Hyperband](/docs/orchestration) strategies
- Schedule recurring jobs with cron, interval, or datetime triggers
- Chain operations with hooks and events for automation

## Sandboxes [#sandboxes]

[Sandboxes](/docs/guides/sandboxes/overview) give engineers and agents an interactive execution surface inside Polyaxon-managed runs — run commands, open a shell, transfer files, and connect over SSH while compute, credentials, artifacts, and audit stay inside Polyaxon.

- Open interactive, reconnectable shells in a live run
- Execute one-off or background commands and stream their output
- Transfer files in and out, and connect local tools over SSH
- Keep credentials, artifacts, permissions, and audit inside the platform

## Model Registry [#registry]

[Model Registry](/docs/registry) provides a centralized catalog for versioning, staging, and promoting models and artifacts across your ML lifecycle.

- Version and promote models through staging, production, and archived stages
- Share reusable components via the [Component Hub](/docs/registry)
- Track artifact versions with full lineage back to training runs

## Why Polyaxon?

- **Open source:** Fully open source with public API for custom integrations
- **Kubernetes native:** Runs on any Kubernetes cluster with GPU support
- **Python SDK:** Native Python client for tracking, managing runs, and API access
- **100+ integrations:** Works with PyTorch, TensorFlow, Keras, Scikit-learn, and many more
- **Distributed training:** Built-in support for Horovod, PyTorch DDP, TensorFlow, and MPI
- **Full platform:** Suite of tools for the complete ML lifecycle from experimentation to deployment
