> **N.B.**: If you are upgrading from polyaxon v1.x, please check the [v2 release notes](/docs/releases/2-0/) and the [migration guide](/docs/resources/migration/#migration-from-v1x-to-v2y).

## V2.16.0

### Core

 * **New**: Add `plugins.tmux` for reconnectable K8s exec sessions via tmux injection.
 * **Fix**: Issue in requirements compatibility issue with traceml version.
 * **Fix**: Fix SDK configuration compatibility with kubernetes client v35 (missing `tls_server_name`).
 * **Fix**: Fix CLI shell inspection parsing to handle the `pods` wrapper in the inspect response.
 * **Enhancement**: Expose tmux plugin state in run `meta_info` for CLI and UI consumption.
 * **Enhancement**: Auto-use tmux session in CLI shell and UI when the tmux plugin is enabled.
 * **Enhancement**: Rewrite CLI `PseudoTerminal` to remove unused fork, fix cleanup, and improve error handling.
 * **Enhancement**: Add tmux auxiliary container configuration to deployment schema.
 * **Enhancement**: Add crons to clean up old stats.

### Deployment

 * **Enhancement**: Improve post-installation instructions.
 * **Enhancement**: Add tmux init container to Helm charts and agent configuration.

### UI

 * **Enhancement**: Improve agent host validation logic.
 * **Enhancement**: Support space-separated commands in the shell component (in addition to comma-separated).
 * **Enhancement**: Auto-detect tmux plugin and set default shell command to tmux when enabled.
